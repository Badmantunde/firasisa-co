## ⚙️ TECHNICAL ARCHITECTURE — STATIC SITE GENERATION (SSG)

**firasisa.com must be built as a Static Site Generated (SSG) website.** This is a non-negotiable architectural decision that directly supports the site's SEO goals, performance benchmarks, and long-term scalability.

### Recommended SSG Framework
**Primary recommendation: [Next.js](https://nextjs.org)** with `output: 'export'` for full static export.  
**Alternative:** [Astro](https://astro.build) — ideal if content-heavy with minimal interactivity needed (excellent Core Web Vitals out of the box).  
**Avoid:** WordPress, Webflow CMS, or server-rendered solutions — they introduce unnecessary overhead and reduce SEO control.

| Framework | Strengths | Best If |
|-----------|-----------|---------|
| **Next.js (SSG)** | React ecosystem, image optimization, ISR option | Team knows React, wants flexibility |
| **Astro** | Near-zero JS by default, fastest possible pages, MDX blog support | Content-first, maximum Lighthouse scores |
| **Gatsby** | GraphQL data layer, mature plugin ecosystem | Complex data sourcing needs |

### Why SSG for firasisa.com

#### 1. SEO Performance — The #1 Reason
SSG produces fully pre-rendered HTML pages. Search engine crawlers receive complete, structured content immediately — no JavaScript execution required. This means:
- Every H1, meta tag, schema block, and keyword lands in the crawler's first pass
- No render-blocking delays that bury keyword-rich content
- Faster indexation of all pages including blog posts and service deep-dives
- Full control over `<head>` metadata per page — critical for the per-page SEO titles and meta descriptions defined in this file

#### 2. Core Web Vitals — Google Ranking Signals
Google's Core Web Vitals (LCP, CLS, INP) are direct ranking factors. SSG consistently achieves:
- **LCP (Largest Contentful Paint):** Under 1.2s — HTML is pre-built, no server wait time
- **CLS (Cumulative Layout Shift):** Near 0 — static assets load predictably
- **INP (Interaction to Next Paint):** Minimal — JavaScript is hydrated selectively, not globally

Target: **Lighthouse score 95+ across all four categories** (Performance, Accessibility, Best Practices, SEO).

#### 3. Hosting — Free, Fast, Global CDN
SSG enables deployment to edge CDN platforms with zero server costs:
- **Vercel** (recommended for Next.js) — global edge, automatic deploys from Git
- **Netlify** — excellent for Astro, built-in form handling, branch previews
- **Cloudflare Pages** — fastest global distribution, free tier generous
- All options support custom domain `firasisa.com` with automatic HTTPS

#### 4. Security
No server = no server attack surface. SSG sites have no database, no PHP, no CMS login endpoint. For a high-profile founder website, a minimal attack surface is a meaningful advantage.

#### 5. Reliability & Uptime
Static files served from a CDN deliver **99.99%+ uptime** with no server maintenance, no database crashes, and no hosting bills that scale with traffic. If a piece of content goes viral, the site will not go down.

### SSG Content Strategy

#### Blog / Writing Section
Blog posts should be authored in **MDX** (Markdown + JSX) files stored in the repository. This enables:
- Git-based content versioning
- Easy addition of new posts without a CMS
- Automatic static page generation per post at build time
- Each post pre-rendered with its own SEO metadata, OG image, and schema markup

**File structure example:**
```
/content
  /posts
    why-i-built-crypto-dispensers.mdx
    cdreload-bitcoin-at-cvs.mdx
    bitcoin-vs-inflation.mdx
```

#### Dynamic-feeling Features (Still Static)
These features can be implemented without a server:
- **Contact form** → Netlify Forms or Formspree (static form submission)
- **Newsletter signup** → ConvertKit or Mailchimp embed
- **Search** → Pagefind (static search index generated at build time)
- **Comments** → Giscus (GitHub Discussions-based, no backend needed)

#### Incremental Updates
If using Next.js, **ISR (Incremental Static Regeneration)** can be used for pages that need occasional freshness (e.g., press mentions, latest blog posts) without sacrificing static-first performance.
